Bedstack is organized as a monorepo using **Bun Workspaces** and **Turborepo**.

## Directory Layout

```bash
.
├── apps
│   ├── conduit     # The RealWorld backend implementation
│   └── www         # This documentation site (Fumadocs)
├── packages        # (Coming soon) Shared code and utilities
├── bin             # CLI tools and scripts
├── docker-compose.yml
└── turbo.json      # Turborepo configuration
```

## `apps/conduit` Structure

The `conduit` app follows a feature-based folder structure:

```bash
apps/conduit/src
├── articles        # Article management (CRUD, favorites, tags)
├── auth            # Authentication logic and middleware
├── comments        # Comment management
├── core            # Core logic and base Elysia setup
├── database        # Drizzle instance and schema definitions
├── profiles        # User profiles and following
├── shared          # Shared utilities and types
├── tags            # Tag management
└── users           # User account management
```

### Key Files in a Feature Folder

Each feature (e.g., `users`) typically contains:

- `*.module.ts`: Elysia controller/router for the feature.
- `*.service.ts`: Business logic and database interactions.
- `*.schema.ts`: ArkType validation schemas.
- `*.test.ts`: Unit and integration tests.

## Documentation (`apps/www`)

The documentation is built with [Fumadocs](https://fumadocs.vercel.app/), a powerful framework for building documentation with Next.js.

- `content/docs`: MDX files for the documentation.
- `src/app`: Next.js App Router components.
- `source.config.ts`: Fumadocs configuration.
